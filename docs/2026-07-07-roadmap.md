# Conn roadmap

Updated 2026-07-13 after the reliability program implementation.

## Where the project is

The reliability program in
`docs/2026-07-12-voice-first-reliability-spec.md` is implemented and
mechanically green: evidence truth, wire and lifecycle integrity, voice-turn
integrity, the semantic intent boundary, capability reporting, targeted
witnesses with adaptive verification, bounded recovery, and realistic
evaluation gates all have pinned tests and recorded runs. The sections below
each carry their standing. Live acceptance against the new signed build and
the human product gate are the open work; `docs/NEXT-SESSION.md` sequences
them.

## Evidence truth first (done 2026-07-13)

Make one accepted PTT turn auditable from key press to user-visible outcome.

Required:

- exact assistant transcript
- correct `app_hotkey` source
- gesture, turn, response, plan, and transaction correlation
- signed build, commit, config, PID, and schema identity
- Swift UI acknowledgements
- per-turn latency distributions
- blocked proposals and final goal outcome in receipts
- one-click sanitized `Report Last Command`
- replay cassette for the July 12 failure session

Done means the logs can explain what the user heard, what Conn tried, why it
failed, and whether anything was dispatched.

## Repair deterministic integrity defects (done 2026-07-13)

Fix the current wire and lifecycle failures before architecture expands.

Required:

- semantic context item lifecycle follows live API limits and server
  acknowledgements
- uncreated items are never deleted
- response cancellation binds an active response ID
- a strict fake Realtime server enforces the live protocol in tests
- app-launched daemon carries an authenticated ownership lease
- graceful app quit shuts down its daemon
- orphaned owned daemon exits after bounded parent-loss grace
- foreign port owners are never killed or adopted

Done means zero normal-path protocol errors across 1,000 replayed turns and 50
quit and reopen cycles without manual cleanup.

## Make voice turns indivisible (done 2026-07-13; live acoustic barge-in check pending)

Required:

- gesture IDs and accepted or rejected PTT acknowledgements
- duplicate modifier edges are idempotent
- playback flushes before a barge-in mic gate opens
- pre-roll captured during playback is suppressed
- short voiced holds are accepted
- silent taps are rejected visibly
- stale island text clears at turn start

Done means 500 PTT cycles with no stuck phase, lost release, duplicate turn, or
silent voiced-turn loss. Recorded playback watermark must never appear in
uploaded command audio.

## Replace model mechanics with semantic intent (done 2026-07-13; live voice runs pending)

The model should say what the user wants. Local code decides how and how to
prove it.

First vertical slices:

- `create(kind=tab)` for Safari New Tab
- `select_relative(relation=next, kind=document)` for Notes navigation

The model does not supply shortcuts, menu paths, snapshot refs, AX actions,
risk, or effect predicates. Conn.app discovers the live affordance, compiles a
bounded plan, and declares its verified, dispatch-only, or blocked ceiling.

Done means the exact failed voice commands complete or ask one useful question
without raw mechanism input from the model.

## Add capability and witness compilation (core done 2026-07-13; menu mark-state witnesses and ref-targeted rereads remain)

Expand the small app-agnostic intent algebra only after the two vertical slices
work.

Required:

- bounded native capability reports
- unique semantic target ranking with ambiguity refusal
- live lazy-menu and shortcut discovery
- compiler-owned causal witnesses
- targeted read sets and adaptive verification backoff
- bridge deadlines aligned to native action budgets
- empirical support history that can only downgrade exposure

No bundle-specific command catalog is allowed.

## Add one safe recovery (done 2026-07-13)

One fresh re-observation and replan is allowed only after proven
`not_dispatched`. Any dispatched or possibly-dispatched outcome stops.

Failures must say what safely happened and what the user can do next. Repeated
identical plan shapes stop after one attempt.

## Replace green-count theater with product evidence (gates built 2026-07-13; live and human runs pending)

Keep current unit and harness tests, but label their scope honestly. Add:

- strict protocol replay
- signed real-AX fixture runs
- prerecorded PTT and audio runs
- opt-in live model intent eval over at least 200 reviewed paraphrases
- top 20 ordinary-command completion corpus
- three 100-turn soak sessions

Daily-driver acceptance requires:

- zero wrong targets
- zero false `Done.`
- zero normal-path protocol errors
- zero manual daemon cleanup
- at least 95 percent first-try completion across supported top-20 commands
- at least 99 percent completion after one safe clarification or replan
- release-to-effect p50 at most 2.0s and p95 at most 4.0s for warm semantic
  actions
- 30 ordinary commands across three work sessions
- at least 90 percent faster than hands or useful hands-free

## Later

Only after the daily-driver gate passes should Conn consider direct API or MCP
planes, visual control, sound, motion polish, character work, or broader
capability expansion.

## Still out of scope

- ScreenCaptureKit action capture
- Vision OCR and image injection
- visual coordinate actions
- separate computer-use model
- broad app command catalogs
- hidden multi-action macros
- online self-modifying policy
- outbound or destructive actions
- sound, motion, MCP, and character work
